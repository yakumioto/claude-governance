# 技能重命名设计文档

## 概述

将技能命名从 `l1/l2/l3` 重命名为 `docs/fix/feat`，提升命令可理解性。

**日期：** 2026-02-25
**状态：** 已批准

---

## 背景

当前 `l1/l2/l3` 命名对用户不够直观，需要学习框架才能理解。新的命名使用动作导向的术语：
- `/docs` - 文档级变更
- `/fix` - 问题修复
- `/feat` - 功能开发

---

## 目标

- 提升命令可理解性
- 用户无需学习框架即可理解命令用途
- 保持技能功能不变

---

## 方案选择

**采用方案 B：全面术语替换**

- 目录重命名：`l1/` → `docs/`，`l2/` → `fix/`，`l3/` → `feat/`
- 全面更新所有文件中的术语
- 术语一致性：用户查看任何文件都能理解

---

## 架构设计

### 新目录结构

```
skills/
├── docs/           # 原 l1/ - 文档级变更
├── fix/            # 原 l2/ - 小变更/修复
├── feat/           # 原 l3/ - 功能变更
├── classify/       # 变更分级
├── explore/        # 需求探索
├── design/         # 设计讨论
├── doc-gen/        # 文档生成
├── execute/        # 执行器
└── commit/         # Git 提交
```

### 术语映射表

| 旧术语 | 新术语 | 说明 |
|--------|--------|------|
| L1 | Docs / Trivial Changes | 文档级变更 |
| L2 | Tiny / Controlled Changes | 小变更/修复 |
| L3 | Full Feature Changes | 功能变更 |
| `/l1` | `/docs` | 文档命令 |
| `/l2` | `/fix` | 修复命令 |
| `/l3` | `/feat` | 功能命令 |

---

## 模块划分

### 1. 目录重命名

- `skills/l1/` → `skills/docs/`
- `skills/l2/` → `skills/fix/`
- `skills/l3/` → `skills/feat/`

### 2. 文档更新

- `CLAUDE.md` - 全局规则
- `README.md` - 框架说明

### 3. 技能内部更新

每个技能的 `SKILL.md` 需要更新：
- 技能描述中的术语
- 流程说明中的命令引用
- 交叉引用其他技能的地方

### 4. 模板更新

`doc-gen/templates/` 中的模板文件需要更新术语。

---

## 执行步骤

1. 使用 `git mv` 重命名目录（保留历史）
2. 更新 `CLAUDE.md` 中的所有术语和命令引用
3. 更新 `README.md` 中的所有术语和命令引用
4. 更新 `skills/classify/SKILL.md` - 输出格式中的命令提示
5. 更新 `skills/execute/SKILL.md` - 命令用法说明
6. 更新 `skills/docs/SKILL.md`（原 l1）- 全面使用新术语
7. 更新 `skills/fix/SKILL.md`（原 l2）- 全面使用新术语
8. 更新 `skills/feat/SKILL.md`（原 l3）- 全面使用新术语
9. 更新 `doc-gen` 模板中的术语

---

## 错误处理

- 使用 `git mv` 保留文件历史，便于回滚
- 逐个技能重命名并验证
- 如果某个技能重命名失败，可单独回滚

---

## 测试策略

1. **结构验证**：确认目录结构正确
2. **内容验证**：检查所有旧术语已替换
3. **功能验证**：确认新命令可被调用
4. **文档验证**：确认 Markdown 渲染正常

---

## 影响范围

- 仅影响命令命名和文档术语
- 不改变技能的核心逻辑和执行流程
- 不影响已存在的治理文档（docs/spec/、docs/tasks/ 等）
