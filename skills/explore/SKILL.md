# 需求探索与澄清（Explore）

本技能用于在变更实施前进行需求探索和澄清，确保对需求的理解准确。

---

## 目标

通过与用户对话，明确以下内容：
- 目标（Goal）：本次变更要达成什么
- 非目标（Non-Goals）：明确不包括什么
- 约束（Constraints）：有哪些限制条件
- 成功标准（Success Criteria）：如何算完成

---

## 执行流程

1. **理解用户需求**
   - 仔细阅读用户的需求描述
   - 提出澄清性问题，每次只问一个问题

2. **探索需求背景**
   - 为什么需要这个变更？
   - 涉及哪些模块/文件？
   - 有什么依赖关系？

3. **明确边界**
   - 哪些是必须做的？
   - 哪些明确不在范围内？
   - 有哪些技术或业务约束？

4. **确认成功标准**
   - 如何验证变更成功？
   - 有什么可测试的指标？

---

## 输出格式

完成探索后，输出以下结构化摘要：

```
## 需求摘要

**目标：**
- ...

**非目标：**
- ...

**约束：**
- ...

**成功标准：**
- ...
```

---

## 注意事项

- 每次只提一个问题，避免同时问多个问题
- 使用用户能理解的语言，避免过多技术术语
- 如果用户描述模糊，主动举例确认
- 如果发现需求超出当前级别边界（如 fix 变成了 feat），及时提示
