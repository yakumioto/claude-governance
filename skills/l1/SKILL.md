# L1 变更约束说明

本技能用于说明 L1 变更的约束和操作方式。

---

## L1 变更定义

L1 变更是指仅修改文档或注释，不改变代码运行行为的变更。

---

## L1 约束规则

满足以下**全部**条件才能作为 L1 处理：

### 允许的修改

- 仅修改文档或注释：
  - README.md
  - docs/guide/**
  - 任意 *.md 文件
  - 代码注释（// 或 /* */）

### 禁止的修改

- 不改变代码运行行为
- 不改变公共接口
- 不改变默认值或配置语义
- 不新增或修改依赖（go.mod / package.json 等）
- 不修改治理文档（docs/spec/、docs/tasks/、docs/changes/）

---

## 处理方式

**L1 变更无需使用技能，请直接描述你需要的修改。**

---

## 提交说明格式

L1 变更的提交说明必须包含：

```
docs: <summary>

What:
- <修改了什么文档>

Why:
- <为什么修改>

Risk:
- <风险说明，L1 通常为 none>

Verify:
- <如何验证，如：Markdown 渲染正常>
```

---

## Done 定义

L1 变更完成的标准：

- Markdown 文档渲染正常
- 示例命令验证通过（如有）
- 提交说明符合格式要求

---

## 注意事项

- 如果需要修改治理文档（docs/spec/、docs/tasks/、docs/changes/），这不属于 L1
- 如果修改涉及代码行为变更，即使很小也应升级到 L2 或 L3
- 当不确定是否属于 L1 时，使用 `/classify` 进行判断
